## ⚑ Decisions made at unresolved forks (flag-once ledger)

⚑ **E-1 tsSec semantics.** mapping-architecture-spec.md:106-108 pins `tsSec` = seconds-since-start; every live writer + persisted row stores Unix epoch. Decision: **stored payload stays Unix epoch** (back-compat, no discriminator exists to tell rebased from epoch rows apart); rebasing happens only inside a future processed `Track`. The spec line should be amended when track.ts lands.

⚑ **E-2 Freeze timing.** "Frozen at log time, never backfilled" was motivated by NDBC's 45-day rolloff. Earth sources (Open-Meteo, SNOTEL) are archival. Decision: **best-effort fetch at save with a short timeout; offline → session saves without conditions, no retry queue this pass.** A later fetch pinned to `occurredAt` against an archival source would be honest, but is deferred rather than silently added.

⚑ **E-3 Conditions payload.** No spec defines the shape. Decision: `conditions?: ConditionsSnapshot` on SessionPayload — open struct, per-domain optional sub-objects (weather/snow/avalanche for Earth), each reading carrying `source`, `tier` (instrument=1, modeled/forecast=3), `fetchedAt`; avalanche additionally `issuedAt`/`expiresAt` so staleness is visible forever. Water/Sky add their own sub-objects — same low-conflict pattern as SessionPayload blocks.

⚑ **E-4 Gear mileage is derived-on-read**, never a stored odometer (outcome-measured ethos). Components are child gear rows (`parentId` → bike); a component accrues from sessions tagging its **parent** from the component's `acquiredAt` forward. Service thresholds are user-set facts displayed descriptively ("612 km — past your 500 km mark"); no alerts, no push.

⚑ **E-5 Crag pin stored inline** as `location? {lat,lng,name?}` on ClimbingBlock, not as a Spot row — the blessed Spot primitive is the right eventual home but is a cross-dimension entity (Water hangs gauges off it); building it unilaterally on this branch invites a merge collision. Promote pin → Spot when Spot lands. Device pin is the session's location fact; an OpenBeta lat/lon is route metadata and never overwrites it.

⚑ **E-6 NP + power series deferred.** No power data exists in the model (no FIT parser, HK power series ingestion not in this scope). Building `np.ts` now = dead code tested against nothing real. Deferred with the cycling power ingestion pass.

⚑ **E-7 GAP model = Minetti energy-cost curve** (published, error band documentable) rather than Strava's proprietary fit. Output labeled estimated. Error band noted in the module doc.

⚑ **E-8 HK-vs-manual dedupe.** wearable-ingestion-spec covers Garmin-vs-Watch only. Decision: HK ingest **skips** a workout whose time window overlaps an existing session of the same modality family (manual is sovereign; the user logged it). Documented in normalizeWorkouts.

⚑ **E-9 eleSource per adapter**: expo-location → `gps`; GPX `<ele>` → `gps` (device unknowable — understate, never overstate); HKWorkoutRoute point altitude → `gps` (Watch fuses baro but Apple doesn't tag it; claiming barometric would fabricate certainty). `barometric` is reserved for sources that declare it: HK `elevationAscended` metadata → `elevationGainSource: 'barometric'` on EnduranceBlock.

⚑ **E-10 Backcountry privacy** = `private?: boolean` on SessionPayload, set true when resort-vs-backcountry flag = backcountry. Single-user app today; the field is the honest placeholder sharing must honor later (payload JSON → no migration).

⚑ **E-11 Migration numbers.** This branch takes **010 (gear)**. dimension/body, /sky, /water exist and the runner is append-only by integer — two branches claiming 010 breaks the merge. Suggested rule for Dylan: earth reserves 010–012; sky/water/body coordinate before merging to main.

#### E4 review fixes (2026-07-08, 10-angle parallel review + self-verification, pre-commit)

The first draft shipped two real domain-modeling bugs neither caught by tests nor by writing careful doc comments — both found by the review, one independently by 3 of the 10 angles:

- **Fabricated outcome on backfill (4x independently confirmed)** — the inverse function backfilled a *specific* guessed outcome (`'redpoint'`) for any pre-E4 row with `sent:true`, and because `SendDraft.outcome` was a required field, resaving that session for ANY reason (even an unrelated field edit) would write the guess back as a permanent fact — a direct violation of "never fabricate" using this same pass's own stated principle. Fixed: `SendDraft.outcome` is now `ClimbOutcome | null`; the inverse never guesses, and build only writes `outcome` to the payload when the user actually chose one.
- **`fell-hung` incorrectly counted as a send (2x independently confirmed)** — `sent: s.outcome !== 'attempt'` treated a fall/hang as a completed send. Fixed via `isSentOutcome()`: only onsight/flash/redpoint/pinkpoint are sends.
- **Case-defense broke ice grades (self-verified after the altitude angle flagged it)** — the original fix for Font's case-sensitivity bug was an unconditional `.toLowerCase()`, which actively breaks WI/AI (ice) scales — they require UPPERCASE, the opposite direction, verified directly against the installed package (`getScale('wi').isType('wi4')` is false, `.isType('WI4')` is true). Fixed by trying the raw string plus both case variants against `isType()` rather than assuming a single direction.
- **`useCragPin.capture()` race (3x independently confirmed)** — the hook's own JSDoc promised "no-op if already locating" but had no reentrancy guard; only the UI's disabled-button prop incidentally prevented it, and state-update timing meant even a `status`-based guard would race. Fixed with a synchronous `useRef` lock (same pattern as `useGpsTracker`'s `subRef.current`), not React state.
- **Sandbag's `getScore` silently fabricates a score for out-of-range grades** ("V44", UIAA "99"/"13" all `isType()=true` but `getScore()=0`, no warning) — this and the Font case bug are two different bugs in the same function; rather than patch both, `parseClimbGrade` dropped scoring entirely and classifies via `isType()` only. This also resolved two simplification findings (dead `score` field with no consumer; `ClimbGradeSystem` as an unsafe-cast duplicate of sandbag's own `GradeScalesTypes` — now a direct type alias).
- Minor: reused `core/src/geo.ts`'s existing `LatLng` type for the crag pin shape instead of three independent inline `{lat,lng}` literals.
- Deferred, not fixed: `useCragPin`/`useGpsTracker`'s duplicated lazy-import+permission boilerplate (real but low-severity; declined to touch the prior-pass file for marginal gain) — worth a shared-helper extraction if a third such hook shows up. The ambient shim's accuracy is comment-asserted, not compiler-checked against sandbag on a future upgrade — accepted risk; `climbGrade.test.ts` provides behavioral (not structural) drift detection.

⚑ **E-15 UK/ice climbing styles not modeled.** `climbGrade.ts`'s style-biased scale priority has no branch for UK technical/adjective grades or ice climbing — if either `ClimbStyle` is ever added, the heuristic needs a matching priority branch, not just a new style label, or UK "5b" will keep silently resolving to French.

- **Date-only dedup could silently discard real data (multiple angles independently)** — the first draft skipped a date if it had ANY existing climbing session, from any source. That would discard a legitimate same-day scenario: a hand-logged gym session in the morning plus an imported BoardLib session that evening. Fixed: dedup is now scoped to (platform, date) — only this platform's own prior imports are checked, so cross-source same-day sessions coexist and a same-file re-import still no-ops.
- **No confirmation before writing to storage** — format detection is a loose header-column match (`date`+`grade` for 8a.nu); an unrelated spreadsheet with those column names would have been silently imported as real climbing history. Fixed: parsing now populates a review summary ("Found N sessions from PLATFORM — Import them?") with explicit Confirm/Cancel buttons; nothing reaches `createObservation` until the user taps Confirm.
- **Style tie-break fabricated a confident label (self-caught after the altitude angle's framing)** — a genuine 8a.nu style-inference tie (0 unambiguous votes either way, the common case for a pure Font-graded bouldering day) defaulted to `'sport'`. Fixed: ties resolve to `'gym'`, the vocabulary the app and `climbGrade.ts` already use for "mixed/unknown discipline" — reusing an existing honest signal instead of inventing a wrong confident one.
- **`crag` conflated with `route` (2x independently confirmed)** — 8a.nu's alias list let a `crag` (climbing area) column stand in for `route` (climb name) on a header ordering crag first. Fixed: removed `crag` from the route aliases; a header with only crag now leaves `route` honestly absent.
- **Outcome-keyword ordering could flip a failure into a fabricated send (2x independently confirmed)** — a compound style value like "Flash attempt" matched `'flash'` before `'attempt'`, recording a failed try as a clean send. Fixed: non-send keywords (`attempt`, `fell`/`hang`) are now checked first.
- **`is_ascent` comparison wasn't trimmed** — inconsistent with the other fields in the same function; a stray-whitespace cell would misclassify a send as a bid. Fixed.
- **Duplicated types/helpers between the two parsers (2x independently confirmed)** — `ImportedSend`/`ImportedSession`, date extraction, attempts-clamping, and the date-grouping tail were each declared twice. Extracted into `src/lib/climbImport/shared.ts`.
- Documented, not changed: `raw` has no UI consumer yet (a deliberate research-mandated audit commitment, not speculative — noted as the one place this pass's judgment differs from E4's `score` cut); the unbounded `listObservations` fetch and sequential unbatched `createObservation` loop are real but low-severity at this app's current single-user scale, and no batch/transaction API exists in the storage layer yet to fix cheaply; bare-`\r`-only line endings aren't handled (neither BoardLib nor 8a.nu emit that format); `boardlib logbook --no-headers` isn't defended against (an opt-in flag this app's own instructions won't tell Dylan to use).

⚑ **E-16 Climbing CSV import decisions.** (1) Idempotency is (platform, date)-scoped, not a full sync — re-importing after hand-editing individual rows in the source export won't pick up corrections; recovery requires deleting the stale session first, and nothing in the UI currently marks which sessions came from import (`source.platform` is written but never displayed). (2) `raw` verbatim audit trail is real infrastructure with no browsing UI yet — worth a session-detail surface once the redesign lands. (3) A generalized "already imported for this civil day" helper now conceptually overlaps `src/lib/healthkit/normalize.ts`'s `isAlreadyImported` (steps/sleep, healthkit-only) — when E6 builds HealthKit workout ingestion, worth deciding whether to generalize both into one shared dedup helper rather than adding a third variant.
