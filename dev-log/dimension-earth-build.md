#### E4 review fixes (2026-07-08, 10-angle parallel review + self-verification, pre-commit)

The first draft shipped two real domain-modeling bugs neither caught by tests nor by writing careful doc comments â€” both found by the review, one independently by 3 of the 10 angles:

- **Fabricated outcome on backfill (4x independently confirmed)** â€” the inverse function backfilled a *specific* guessed outcome (`'redpoint'`) for any pre-E4 row with `sent:true`, and because `SendDraft.outcome` was a required field, resaving that session for ANY reason (even an unrelated field edit) would write the guess back as a permanent fact â€” a direct violation of "never fabricate" using this same pass's own stated principle. Fixed: `SendDraft.outcome` is now `ClimbOutcome | null`; the inverse never guesses, and build only writes `outcome` to the payload when the user actually chose one.
- **`fell-hung` incorrectly counted as a send (2x independently confirmed)** â€” `sent: s.outcome !== 'attempt'` treated a fall/hang as a completed send. Fixed via `isSentOutcome()`: only onsight/flash/redpoint/pinkpoint are sends.

- **Sandbag's `getScore` silently fabricates a score for out-of-range grades** ("V44", UIAA "99"/"13" all `isType()=true` but `getScore()=0`, no warning) â€” this and the Font case bug are two different bugs in the same function; rather than patch both, `parseClimbGrade` dropped scoring entirely and classifies via `isType()` only. - Minor: reused `core/src/geo.ts`'s existing `LatLng` type for the crag pin shape instead of three independent inline `{lat,lng}` literals.
- Deferred, not fixed: `useCragPin`/`useGpsTracker`'s duplicated lazy-import+permission boilerplate (real but low-severity; declined to touch the prior-pass file for marginal gain) â€” worth a shared-helper extraction if a third such hook shows up.
